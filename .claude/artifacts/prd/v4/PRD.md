## 1. 개요 (Executive Summary)

Maththera는 난산증(Dyscalculia) 아동을 대상으로 한 AI 기반 수학 인지 훈련 시스템이다. 기존 수학 교육 도구가 "맞았다/틀렸다"만 판별하는 한계를 넘어, **오답의 인지적 원인을 자동 분류하고 적응형 훈련을 처방**하는 것이 핵심 가치이다.

### 핵심 차별점

| 기존 접근 | Maththera |
|----------|-----------|
| 정오답 이분법 | 인지 도메인별(전략/공간/주의력) 분리 진단 [research/CLM-002] |
| 정답 = 통과 | 정답이어도 반응 시간(RT)과 전략 효율성 분석 [biz/CLM-002] |
| 난이도 단순 조절 | W-Score(작업기억 부하) + C-Tag(교육과정) 이중 태그 기반 정밀 조절 [pedagogy/CLM-W-Score] |
| 범용 피드백 | 3문장/10자 이내, 비난 금지, 4가지 상황별 표준 프로토콜 [biz/CLM-007] |
| 단일 원인 추정 | 변인 통제 Probe로 복합 원인 정밀 감별 [biz/CLM-005] |

### 솔루션 요약

Master Matrix의 **5단계 오답 진단 파이프라인**(문제 정의 -> 관찰 기록 -> 오답 원인 -> 훈련 매핑 -> 훈련 모듈)을 통해, Cogassist Agent가 매트릭스 체계에 따라 자율적으로 오답 원인을 분류하고 최적의 훈련 경로를 제시한다. [biz/CLM-001, pm/CLM-003]

15,053개 문제로 구성된 W*C 이중태그 문제 뱅크를 기반으로, 규칙 기반 결정 트리와 상태 머신으로 적응형 알고리즘을 구현한다. 복잡한 ML 모델 없이 교육적 일관성과 치료적 재현성을 확보한다. [tech/CLM-001, tech/CLM-003]

## 2. 문제 정의

### 2.1 핵심 문제

난산증은 지능이나 교육 기회의 부족이 아니라, **후두정엽(수 표상)과 전전두엽(집행 제어) 사이의 신경학적 연결성 결함**에서 기인하는 발달적 장애이다. [research/CLM-001] 전 세계 학령기 아동의 3~6%에 영향을 미치며, 난독증(Dyslexia)과 유사한 유병률을 보이지만 사회적 인지도와 전문 개입 도구 모두 현저히 부족하다.

### 2.2 인과 체인

```
수감각 결손 -> 수학능력 저하 -> 난산증 고착
```

이 인과 체인에 대한 가설: **AI 진단-개입으로 수감각 회복이 가능하다.** [biz/CLM-010]

### 2.3 기존 접근의 한계

| 기존 접근 방식 | 한계 |
|--------------|------|
| 문제 반복 풀이 | "왜 틀렸는가"를 설명하지 못함 |
| 난이도 단순 조절 | 인지적 원인 분석 없이 양적 조절만 수행 |
| 범용 적응형 학습 | 난산증 특화 진단 체계 부재 |
| 오프라인 치료 | 접근성, 비용, 전문가 수 제한 |

### 2.4 인지적 프레임워크: C-S-E 모델

Maththera의 진단 체계는 C-S-E 3축 인지 모델에 기초한다. [research/CLM-002, pedagogy/CLM-CSE]

| 축 | 의미 | 뇌 영역 | 상세 |
|----|------|---------|------|
| **C (Concept)** | 장기기억의 수 표상 | 후두정엽 | C1(기수성) ~ C6(가역성) 6단계 위계. 하위 결함이 상위 전체를 무너뜨리는 누적 구조 |
| **S (Strategy)** | 문제 해결 설계 | 전전두엽 | S1(전부 세기) ~ S7(유형 분류). 각 S는 대응 C에 1:1 의존 [research/CLM-003] |
| **E (Execution)** | 작업기억 처리 부하 | 작업기억 | E1(정보유지량) x E2(절차단계) x E3(수크기) [research/CLM-004] |

난산증 아동은 E축(작업기억) 임계치가 낮아, 같은 문제라도 정상 아동 대비 더 큰 인지적 압박을 경험한다. [pedagogy/CLM-E-WM]

## 4. 대상 사용자 / 페르소나

|---------|------|-----------------|
| 진단된 난산증 아동 | 전문기관 진단 | 약 12~24만명 (학령기 400만 x 3~6%) |
| 수학 학습부진 아동 | 미진단이나 심각한 어려움 | 약 40~60만명 (학령기 10~15%) |
| 수감각 강화 필요 아동 | 예방적 훈련 대상 | 약 80~120만명 (학령기 20~30%) |

| 구매자 | 니즈 | 채널 |
|-------|------|------|
| 학부모 | 자녀의 수학 불안 해소, 전문 개입 접근 | B2C (앱스토어, 웹) |
| 특수교육 교사 | 개별화 교육 도구, 진단 보조 | B2B (학교/교육청) |
| 소아정신과/임상심리사 | 과학적 근거 기반 훈련 도구 | B2B (병원/치료센터) |
| 학습치료센터 | 비용 효율적 치료 콘텐츠 | B2B (센터 라이선스) |

### 4.3 아동 페르소나

**"민준이" (8세, 초등 2학년)**
- 지능은 평균 이상이나 수학 시간만 되면 움츠러듦
- 8+6 문제를 손가락으로 하나씩 세어 14를 구함 (정답이지만 12초 소요)
- "나는 수학이 싫어"가 입버릇
- 수학 불안(Math Anxiety)이 높아 틀릴까봐 시도를 회피
- Maththera가 해결: 전략 미숙(MAKE10 부재)을 진단하고, '틀려도 괜찮다'는 분위기에서 10의 보수 전략을 단계적으로 훈련 [biz/CLM-002, biz/CLM-006]

## 5. 기능 요구사항

### FR-001: Master Matrix 오답 진단 파이프라인

사용자가 문제를 풀면 Matrix가 **5단계 파이프라인**을 자동 실행한다. [pm/CLM-003]

1. **문제 정의**: L-level, Cur-C level, 전략 태그(STR_MAKE10, STR_PLACEVALUE 등) 자동 설정
2. **관찰 기록**: RT(밀리초 단위), 행동 태그(OBS_*) 수집
3. **오답 원인**: 관찰 기록과 문제 정의를 교차 분석하여 오답 원인 태그 결정
4. **훈련 매핑**: 오답 원인 태그에 따라 훈련 ID 자동 매핑
5. **훈련 모듈**: 매핑된 훈련 ID에 해당하는 인터랙티브 훈련 콘텐츠 제시

**수용 기준**:
- AC-001-1: 모든 문제에 L-level, Cur-C level, 전략 태그가 정확히 태깅된다
- AC-001-2: RT가 밀리초 단위로 수집되며 10ms 이하 오차를 보장한다
- AC-001-3: 5단계 파이프라인이 2초 이내에 완료된다

### FR-002: 전략 불일치 진단 (유스케이스 1)

정답이더라도 RT가 기준보다 느리면 전략 미숙으로 진단한다. [pm/CLM-004, biz/CLM-002]

| 입력 | 진단 | 처방 |
|------|------|------|
| 8+6 = 14 (정답), RT 12초 | STR_MAKE10 요구 vs OBS_OVERRELIANCE_COUNTING 관찰 | T12(10짝찾기) -> T17(10만들어더하기) |

**핵심**: "정답이어도 느리면 자동화 미완성"이라는 판단은 기존 에듀테크에 없는 차별화. [research/CLM-006]

### FR-003: 공간 인지 결손 진단 (유스케이스 2)

오답 값이 특정 자릿수만 계산한 결과와 일치하면 SPATIAL 영역의 인지 문제로 진단한다. [pm/CLM-005, research/CLM-007]

| 입력 | 진단 | 처방 |
|------|------|------|
| 23+4 = 6 (오답) | 오답(6) = 일의 자리(4)를 십의 자리(2)에 더한 결과 | OBS_PLACEVALUE_CONFUSION -> T26(자릿값이해) |

**핵심**: 동일 오답이라도 산술적 실수와 인지 구조 결손을 감별한다.

### FR-004: 주의력/충동성 진단 (유스케이스 3)

L < C(아동 레벨 < 문제 복잡도) 상황에서 극도로 빠른 RT와 무작위 응답을 감지하면, 수학적 지식이 아닌 ATTENTION 도메인 개입을 우선한다. [pm/CLM-006, research/CLM-008]

| 입력 | 진단 | 처방 |
|------|------|------|
| 18+6, RT 0.8초, 무작위 응답 | OBS_VERY_FAST_RT + OBS_RANDOM_GUESSING | ATTENTION_L0_IMPULSE -> T30(주의력환기) 또는 T19(단계별힌트) |

**핵심**: 난산증과 ADHD 공존 장애(comorbidity) 감별의 임상적 근거. [biz/CLM-004]

### FR-005: 변인 통제 Probe 시스템 (유스케이스 4)

오답 원인이 2가지 이상 추정될 때, 변인을 통제한 확인용 문제(Probe)를 즉시 출제하여 원인을 확정한다. [pm/CLM-007, biz/CLM-005]

| 입력 | Probe | 판정 |
|------|-------|------|
| 28+15 = 33 (오답, 정답 43과 -10 차이) | 20+15 (받아올림 변인 C3->C1 제거) | Probe 오답: SPATIAL_L4_ALIGNMENT -> T26 |
| | | Probe 정답: STRATEGY_L4_PLACEVALUE -> T27 |

**핵심**: 과학적 실험 설계(변인 통제) 기반의 정밀 감별 진단 자동화. [research/CLM-009]

### FR-006: 적응형 알고리즘 (Adaptive Logic)

W-Score(작업기억 부하)와 C-Tag(교육과정)의 이중 태그를 기반으로 한 적응형 문제 출제. [tech/CLM-001, tech/CLM-002]

**2경로 분기 구조:**

| 경로 | 조건 | 동작 | 예시 |
|------|------|------|------|
| Route A (W 하향) | 전략은 알지만 계산에서 막힘 | C 레벨 유지, W_총점 낮은 문제 검색 | 8+6(W:4.5) 실패 -> 9+2(W:2.5) |
| Route B (C 하향) | 개념 자체를 모름 | C 레벨 1~2단계 하향 | 8+6(C:7) 실패 -> 8+□=10(C:5) |

**4가지 반응 패턴 분기:** [pedagogy/CLM-4-Pattern, tech/CLM-003]

| 반응 | C 조정 | W 조정 | 의미 |
|------|--------|--------|------|
| 심각한 오답 (8+6=1) | C -2 | W < 1.0 | 수 관계 기초 복구 |
| 논리적 오답 (가르기 실수) | C 유지 | W -1.5 | 작업기억 부하 경감 |
| 정답+느림 (5초+) | C 유지 | 동일 수준 다른 숫자 | 자동화(Fluency) 훈련 |
| 정답+빠름 | C+1 또는 유지 | W +1.0 | 도전적 과제 |

**3단계 복귀 루프 (Scaffolded Recovery):** [tech/CLM-006]

1. 기준점 고정: 단순화된 문제(예: 100+90)로 큰 수 결합 인지
2. 미세 조정(가르기): "원래 문제는 100이 아니라 98이었지? 190에서 2를 빼면?"
3. 복귀: 원래 문제(98+90) 재제시, 아동 스스로 '어디서 가를지' 결정 확인

**Bridge Loop:** [pedagogy/CLM-Bridge]

쉬운 문제(100+90=190)는 풀지만 원래 문제(98+90)로 돌아가면 막히는 경우, 빌려오기(Borrowing) 전략으로 두 문제를 교량 연결: 98->100 만들기 -> 90-2=88 -> 100+88=188.

**White-out 현상:** [pedagogy/CLM-Whiteout]

100+90도 모르면 10+9로 대폭 후퇴. 최후의 보루는 9+1=10 (C=5단계).

### FR-007: AI 피드백 시스템

'따뜻하고 격려하는 친구 같은 튜터' 페르소나를 일관 유지한다. [pm/CLM-008, pm/CLM-011, biz/CLM-006]

**기본 규칙:**
- 전체 3문장 이내
- 문장당 10자 이내 (수식/힌트 필요 시 최대 15자)
- 반말 사용 (존댓말 금지)
- 비난/단정 금지
- 힌트는 1개만
- 항상 다음 한 걸음 제시

**상황별 템플릿:** [pm/CLM-009, biz/CLM-008]

| 상황 | 구성 | 예시 |
|------|------|------|
| 정답 | 칭찬 1 + 확인 1 | "좋아!" / "정답이야!" |
| 오답 | 공감 1 + 힌트 1 + 재도전 1 | "괜찮아." / "하나만 더!" / "다시 해볼까?" |
| 과정 실수 | 칭찬 1 + 포인트 1 + 재시도 1 | "거의 다 왔어!" / "자리만 봐!" |
| 무응답 | 안심 1 + 작은 단계 1 + 선택지 1 | "괜찮아." / "하나씩 하자." |

**금지 목록:** [pm/CLM-010, research/CLM-014]
- 금지 표현: "틀렸어", "아니야", "왜 못해", "정신 차려", "빨리", "당연히", 비꼼/평가
- 금지 방식: 3문장 초과, 힌트 2개 이상, 비교/압박/벌점 느낌

**힌트 원칙:** [biz/CLM-009]
- 정답을 직접 말하지 않기
- 아이가 할 수 있는 작은 행동으로 만들기 ("손가락 써봐", "하나 더해봐")
- ZPD(근접발달영역) 기반 scaffolding과 일치

**구현 방식:** [tech/CLM-009]
- 템플릿 기반 피드백 시스템 (LLM 자유 생성 금지)
- 사전 검수된 템플릿 풀에서 상황별 선택, 변수 슬롯(숫자/연산자)만 동적 충전
- 금지어 필터(정규식 + 블랙리스트)로 후처리 검증

### FR-008: W-Score 부하 시스템

각 문제의 작업기억 부하를 6개 차원으로 정량화한다. [pedagogy/CLM-W-Score, pedagogy/CLM-W-Dims]

| 차원 | 의미 | 규칙 |
|------|------|------|
| W1_연산 | 연산 유형 | 덧셈 0, 뺄셈 +1.5 |
| W2_자릿수 | 자릿수 | 한자리 0, 두자리 +3, 두자리x두자리 +4 |
| W3_숫자크기 | 수의 크기 | 6이상 +0.5, 양쪽 큰 수 +1.0 |
| W4_복잡성 | 절차 복잡도 | 받아올림/받아내림 +3.0 |
| W5_특수수 | 보정 | 같은 수 -0.5, 10배수 -1.0 |
| W6_표현 | 표현 형식 | 가로셈 +0.5, 세로셈 0 |

- W_총점 범위: 0~9.5 (하한 0으로 클램핑)
- W 레벨: 0.5 단위 구간화 -> 1~23 레벨

### FR-009: C-Tag 교육과정 체계

교육과정 단원(Cur-C)과 인지진단 개념 태그(Cog-C)를 분리 운영한다. [pedagogy/CLM-C-Tag, pedagogy/CLM-Cog-C]

**교육과정 단원 (Cur-C1~Cur-C15):**

| 단원 | 내용 |
|------|------|
| Cur-C1 | 한자리 덧셈 |
| Cur-C2 | 한자리 뺄셈 |
| Cur-C3 | 0 더하기 |
| Cur-C4 | 0 빼기 |
| Cur-C5 | 더해서 10 |
| Cur-C6 | 10에서 빼기 |
| Cur-C7 | 십몇 덧셈 |
| Cur-C8 | 받아올림 없는 두자리 덧셈 |
| Cur-C9 | 받아내림 없는 두자리 뺄셈 |
| Cur-C10~C12 | 받아올림 덧셈 |
| Cur-C13~C15 | 받아내림 뺄셈 |

**인지진단 개념 태그 (Cog-C1~Cog-C6):**

| 태그 | 개념 | 전략 매핑 |
|------|------|----------|
| Cog-C1 | 기수성 (Cardinality) | S1 (전부 세기) |
| Cog-C2 | 수의 크기 비교 (Magnitude) | S2 (큰 수에서 세기) |
| Cog-C3 | 수 감각 (Number Sense) | S3 (자동 인출) |
| Cog-C4 | 자릿수 원리 (Place Value) | S4 (자릿수 분리) |
| Cog-C5 | 결합 및 분해 (Composition) | S5 (분해 합산/10 만들기) |
| Cog-C6 | 연산의 가역성 (Invertibility) | S6 (역연산) |

**개념 태그 판단 상태 4단계:** [pedagogy/CLM-State-4]

| 상태 | 의미 |
|------|------|
| Normal | 정상 (데이터 유효) |
| Distorted | 오류 (데이터 오염) |
| Incomplete | 부분 누락 (데이터 손실) |
| Void | 치명적 오류 (데이터 부재) |

## 7. UX 노트

### 7.1 보이스 톤 가이드

AI 피드백의 궁극적 목표는 아이가 **'틀려도 괜찮다'**는 느낌으로 다시 시도하게 만드는 것이다. [marketing/CLM-010, biz/CLM-006]

이 가이드는 난산증 아동의 **수학 불안(Math Anxiety)**과 **학습된 무력감(Learned Helplessness)**을 경감시키는 안전장치로 설계되었다. [research/CLM-013, research/CLM-014]

### 7.2 시각 보조 체계

기찻길-블록 통합 시각 메타포를 통해 추상적 수 개념을 공간적-물리적 경험으로 전환한다 (체화된 인지, Embodied Cognition). [research/CLM-012, pedagogy/CLM-Visual]

| 단계 | 메타포 | 교육적 의미 |
|------|--------|------------|
| 1단계 (기초) | 기찻길과 낱개 블록 | 일대일 대응 (Cog-C1) |
| 3단계 (기준) | 5번 중간역, 색상 변화 | 기준점 인식 (Cog-C3) |
| 4단계 (단위) | 황금 기차 변신 (10개 묶음) | 자릿값 원리 (Cog-C4) |
| 5단계 (연산) | 5-10 채우기 작전 | 결합/분해 (Cog-C5) |
| 6단계 (검토) | 기차 후진 | 가역성 (Cog-C6) |

### 7.3 Downscaling/Upscaling 전략과 UI

**Downscaling 4전략:** [tech/CLM-008]
1. 시각적 보조(10 프레임/계란판) 투입
2. 단계 분리 ("계산하지 말고 결정만 해봐")
3. 숫자 크기 축소
4. 세로셈 전환

**Upscaling 3전략:**
1. 시각 힌트 제거
2. 추상화 (가르기 선 없이)
3. 수치 확장

### 7.4 진단 질문 입력 방식

MVP에서는 **객관식(2~3개 선택지)** 방식을 채택한다. 초등 저학년 아동이 자유 텍스트로 응답하는 것은 현실적으로 어렵기 때문이다. Phase 2에서 STT+NLU 기반 음성 인식으로 확장한다. [tech/CLM-007]
