## 1. 훈련 설계 체계 분석

### 난산증의 신경학적 정의

난산증은 지능과 교육 기회가 충분함에도 불구하고, 수적 정보를 처리하는 뇌 회로의 발달적 결함으로 인해 수학 학습에 심각한 어려움을 겪는 상태이다. 후두정엽(수 표상)과 전전두엽(집행 제어) 사이의 신경학적 연결성 문제이다.
[출처: 난산증/CSE 인지 모델/chunk-0000]

### CSE 프레임워크: 3축 인지 분석

학습자의 상태를 **개념(C)**, **전략(S)**, **집행(E)** 세 축으로 분석한다:

- **C (Concept)**: 장기기억의 수 표상 (Data)
- **S (Strategy)**: 전전두엽의 문제 해결 설계 (Control)
- **E (Execution)**: 작업기억의 처리 부하 (Load)

[출처: 난산증/CSE 인지 모델/chunk-0001]

### 전략-개념 1:1 매핑 구조

전략(S) 체계는 S1(전부 세기)~S7(유형 분류)의 7단계로, 각각 C1~C6 개념과 1:1 매핑된다:
- S1-C1(기수성)
- S2-C2(크기 비교)
- S3-C3(수 감각)
- S4-C4(자릿수)
- S5-C5(분해)
- S6-C6(가역성)
- S7은 C1~C6 통합

[출처: 난산증/CSE 인지 모델/chunk-0003]

### 시각 메타포 체계: 기찻길-블록 모델

| 단계 | 메타포 | 교육적 의미 |
|------|--------|------------|
| 1단계 (기초) | 기찻길과 낱개 블록 | 일대일 대응 (C1) |
| 3단계 (기준) | 5번 중간역과 색상 변화 | 기준점 인식 (C3) |
| 4단계 (단위) | 황금 기차 변신(10개 묶음) | 자릿값 원리 (C4) |
| 5단계 (연산) | 5-10 채우기 작전 | 결합/분해 (C5) |
| 6단계 (검토) | 기차 후진 | 가역성 (C6) |

[출처: 난산증/CSE 인지 모델/chunk-0006]

## 2. W-Score 부하 시스템 정의

### W-Score 6차원 체계

Maththera의 W-Score 시스템은 **6개 차원(W1~W6)**으로 문제의 작업기억 부하를 정량화한다. W_총점 범위는 약 0~9.5이며, 이를 0.5 단위로 구간화하여 **W 레벨 1~23**으로 변환한다.
[출처: 훈련로직-Path-Tag (W*C 이중태그 15,053문제)/chunk-0000, W/C 이중 태그 문제 뱅크/chunk-0000]

### 6개 차원 상세 정의

| 차원 | 가중치 규칙 | 예시 |
|------|-----------|------|
| **W1_연산** | 덧셈 0, 뺄셈 +1.5 | 3+2=0점, 5-2=+1.5점 |
| **W2_자릿수** | 한자리 0, 두자리 3, 두자리x두자리 4 | 5+3=0점, 15+3=3점, 15+23=4점 |
| **W3_숫자크기** | 6이상 +0.5, 양쪽 큰 수 +1.0 | 3+4=0점, 7+8=+1.0점 |
| **W4_복잡성** | 받아올림/받아내림 +3.0 | 5+3=0점, 7+6=+3.0점 |
| **W5_특수수** | 같은 수 -0.5, 10배수 -1.0 | 5+5=-0.5점, 10+20=-1.0점 |
| **W6_표현** | 가로셈 +0.5, 세로셈 0 | 세로 배치=0점, 가로 배치=+0.5점 |

[출처: W/C 이중 태그 문제 뱅크/chunk-0000]

### 작업기억 3요소 (E축)

집행(E) 영역에서 작업기억은 3가지 요소로 세분화된다:
- **E1 (정보 유지량)**: 자릿수
- **E2 (절차적 단계)**: 받아올림 유무
- **E3 (수의 물리적 크기)**: Low (0~4), Mid (5~7), High (8~9)

난산증 아동은 이 부하 임계치가 낮다.
[출처: 난산증/CSE 인지 모델/chunk-0004]

### W*C 이중태그 문제 뱅크

15,053문제로 구성되며, 각 문제에 W1~W6 가중치, W_총점, W 레벨(1~23), C 단원(1~15)이 태깅되어 있다. W_총점 범위는 0(예: 0+0)에서 11(예: 99+98)까지 분포한다.
[출처: 훈련로직-Path-Tag (W*C 이중태그 15,053문제)/chunk-0000, chunk-0150]

## 3. C-Tag 교육과정 매핑

### 교육과정 C단원 (C1~C15)

C-Tag 교육과정 체계는 **C1~C15의 15개 단원**으로 구성된다:

| 단원 | 내용 | 비고 |
|------|------|------|
| C1 | 한자리 덧셈 | |
| C2 | 한자리 뺄셈 | |
| C3 | 0 더하기 | |
| C4 | 0 빼기 | |
| C5 | 더해서 10 | |
| C6 | 10에서 빼기 | |
| C7 | 십몇 덧셈 | |
| C8 | 받아올림 없는 두자리 덧셈 | |
| C9 | 받아내림 없는 두자리 뺄셈 | |
| C10~C12 | 받아올림 덧셈 | |
| C13~C15 | 받아내림 뺄셈 | |

[출처: W/C 이중 태그 문제 뱅크/chunk-0000]

### 교육과정 vs 인지진단 체계 분리

**인지진단 Concept Tag(C1~C6)**는 교육과정 C단원(C1~C15)과 **별개의 체계**이다:

**인지진단 6가지 개념**:
- C1: 기수성 (Cardinality)
- C2: 수의 크기 비교 (Magnitude)
- C3: 수 감각 (Number Sense)
- C4: 자릿수 원리 (Place Value)
- C5: 결합 및 분해 (Composition & Decomposition)
- C6: 연산의 가역성 (Invertibility)

이 6가지 개념으로 아이의 수감각 결함을 진단한다.
[출처: Maththera Concept Tag/chunk-0000, 난산증/CSE 인지 모델/chunk-0002]

## 4. 수감각 진단 및 경로 결정

### Concept Tag 활성화 규칙 (Bottom-up Selection)

Concept Tag의 활성화는 문제 유형별 Trigger Rule에 의해 결정된다. 하위 개념 문제(예: 3+0)에서는 하위 컨셉 태그(C1, C2)만 스캔하는 것이 타당하다.
[출처: Maththera Concept Tag/chunk-0003, chunk-0005]

### 개념 태그 판단 상태 4단계

| 상태 | 의미 | 데이터 상태 |
|------|------|-----------|
| Normal | 정상 | 데이터 유효 |
| Distorted | 오류 | 데이터 오염 |
| Incomplete | 부분 누락 | 데이터 손실 |
| Void | 치명적 오류 | 데이터 부재 |

[출처: Maththera Concept Tag/chunk-0002]

### C5 이원화 제안

C5(결합/분해)의 이원화가 제안되었다:
- **C5a (합성)**: 3과 2를 보고 5를 떠올리는 능력
- **C5b (분해)**: 10을 보고 7과 3으로 가르는 능력

또한 C1-zero(0의 특수성)와 C7 등가성(보류) 태그도 검토 중이다.
[출처: Maththera Concept Tag/chunk-0004]

## 5. Adaptive Logic 알고리즘 분석

### Route A: W Downscaling (작업기억 부하 감소)

전략은 알지만 계산에서 막힐 때 적용된다. 동일한 C 레벨 내에서 W_총점이 낮은 문제를 검색하여, **구조는 유지하되 뇌가 처리해야 할 정보량을 줄여** 성공 경험을 제공한다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0001]

### Route B: C Downscaling (개념 단계 하향)

가르는 이유나 보수 자체를 모를 때 적용된다. C 레벨을 1~2단계 아래로 낮추어, **기초 체력(거리 감각)부터 다시 재활**한다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0001]

### 4가지 반응 패턴 분기

훈련 루프는 아동의 반응에 따라 4가지 경로로 분기한다:

| 패턴 | C 조정 | W 조정 | 의미 |
|------|--------|--------|------|
| (1) 심각한 오답 | C-2 | W<1.0 | 개념 부재, 대폭 후퇴 |
| (2) 논리적 오답 | C 유지 | W-1.5 | 실수, 난이도만 낮춤 |
| (3) 정답+느림 | C 유지 | 동일 수준 다른 숫자 | 반복 훈련 |
| (4) 정답+빠름 | C+1 또는 유지 | W+1.0 | 단계 상승 |

[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0002]

## 6. 훈련 루프 설계

### Bridge Loop: 빌려오기 전략

아이가 쉬운 문제(예: 100+90=190)는 풀지만 원래 문제(98+90)로 돌아가면 다시 막힐 때, **'빌려오기(Borrowing)' 전략**을 통해 두 문제를 연결하는 교량 역할을 한다.

**3단계 분해 프로세스**:
1. 98->100으로 만들기
2. 90-2=88
3. 100+88=188

반복 확인 후 원래 문제로 복귀한다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0007]

### Bridge Loop 실패의 원인

연결 실패는 수감각 요소 중 **가역성(Invertibility)**과 **결합 및 분해(Composition)**의 연결 부위가 느슨할 때 발생하는 현상이다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0007]

### 98+90=100 시나리오 훈련 경로

양적 관계의 '방향성(더하면 커진다)'이 회복된 상태라면, 다음 단계에서는 **'기준수(100)를 넘어서는 양의 확장'**과 **'덩어리째 더하기'**를 연습시킨다. 추천 문제는 **100+90**이다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0005]

### White-out 현상: 최후 보루

100+90도 모르면 **'화이트아웃(White-out)' 현상**으로 판단하고, **10+9**로 대폭 후퇴한다. 10+9가 직관적으로 나오지 않으면 100+90은 절대 풀 수 없기 때문이다. 최후의 보루는 **9+1=10 (C=5단계)**이다.
[출처: 데이터 기반 오답 교정 알고리즘 (Adaptive Logic)/chunk-0006]

## 7. 미해결 질문 (Open Questions)

1. **C5 분리의 W-Score 영향**: C5(결합/분해)의 C5a(합성)와 C5b(분해) 분리가 실제 구현에 반영될 경우, W-Score 가중치 체계에도 변경이 필요한지 검토 필요

2. **C7 등가성 태그 도입**: C7 등가성(Equality) 태그의 도입 여부가 보류 상태 — 등호의 균형 개념이 적응형 알고리즘의 진단 경로에 어떤 영향을 줄지 미결

3. **Level Tag 매핑 불명확**: Level Tag 체계(Level 1~N)와 W 레벨(1~23)의 정확한 매핑 기준이 명시적으로 문서화되지 않음

4. **Bridge Loop 단원 불일치**: Bridge Loop의 실전 적용 시, 아이가 '빌려오기' 단계 중 90-2=88에서 막히는 경우의 C downscaling 경로가 C=6(10에서 빼기)으로 설정되었으나, 90-2는 C=9(받아내림 없는 두자리수 뺄셈)에 해당하여 단원 매핑 불일치 가능성 있음

5. **Voice Tone Guide 충돌**: Voice Tone Guide의 '3문장 이내, 문장당 10자 이내' 제약이 Adaptive Logic의 진단 질문(예: '98은 이미 100이랑 아주 가까운 숫자야...')과 충돌 가능 — 진단 질문용 별도 톤 가이드 필요

## 8. 리스크 (Risks)

| 리스크 | 설명 | 영향도 |
|--------|------|--------|
| **음수 W_총점** | W-Score의 W5(특수수) 감점 규칙이 음수 W_총점을 생성할 수 있음 (예: 0+5의 W_총점이 -0.5). 음수 W_총점의 의미와 처리 방법이 명확하지 않음 | Medium |
| **뺄셈 처방 로직 미비** | 15,053문제 뱅크에서 뺄셈 문제의 C단원 분류가 덧셈과 독립적으로 운영되나, Adaptive Logic의 C downscaling은 덧셈 중심으로 설계되어 뺄셈 경로의 처방 로직이 상대적으로 미비함 | High |
| **C 명명 충돌** | 인지진단 Concept Tag(C1~C6)와 교육과정 C단원(C1~C15)의 명명 충돌 — 동일한 'C' 접두사를 사용하여 시스템 구현 시 혼동 가능 | High |
| **판단 상태 임계값 미정의** | Concept Tag 판단 상태(Normal/Distorted/Incomplete/Void)의 구체적 임계값이 정의되지 않아, AI 에이전트의 자의적 판단 리스크가 잔존 | Medium |
