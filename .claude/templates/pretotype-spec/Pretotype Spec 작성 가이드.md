# Pretotype Spec 작성 가이드 v1.0

시장 반응을 실험으로 검증한 데이터 기반 의사결정 문서

* **핵심 질문:** "이 아이디어에 시장이 실제로 반응하는가?"  
* **프로세스 위치:** Stage 2 \- Pretotype (2주)  
* **Gate 연결:** Gate 2 (Kill Gate \- Go/Pivot/Kill)  
* **작성:** PM (AI 80% 실험 설계·분석 \+ PM 20% 현장 실행·판단)  
* **데이터 수준:** 1차 데이터 (실험 결과). Stage 1의 가설을 행동 데이터로 검증.

## **섹션 맵**

| \# | 섹션명 | 핵심 질문 | 필수/선택 | 작성 시점 |
| ----- | ----- | ----- | ----- | ----- |
| 1 | XYZ 가설 | 무엇을 검증하는가? | 필수 | S2 시작 |
| 2 | 실험 설계 | 어떻게 검증하는가? | 필수 | S2 시작 |
| 3 | 결과 데이터 | 시장이 어떻게 반응했는가? | 필수 | S2 종료 |
| 4 | NPQ 재판정 | Painkiller인가, Vitamin인가? | 필수 | S2 종료 |
| 5 | Go/Pivot/Kill 판정 | 다음 단계로 가는가, 죽이는가? | 필수 | S2 종료 |

**Before/After 구조:** §1\~§2는 실험 전(설계), §3\~§5는 실험 후(결과+판정). 이 문서는 2주에 걸쳐 완성된다.

## **1\. XYZ 가설**

Business Spec §6.3에서 설계한 가설을 실험 실행 가능한 수준으로 구체화한다. Stage 1에서는 "검증 계획"이었다면, 여기서는 "실험 가설"이다.

### **1-1. 핵심 가설**

**"적어도 X%의 Y(타겟)는 Z(행동)할 것이다"**

| 가설 | X% | Y (타겟) | Z (행동) | 성공 기준 | 출처 |
| ----- | ----- | ----- | ----- | ----- | ----- |
| 1차 가설 | \[%\] | \[타겟\] | \[행동\] | \[Go/Kill 경계선\] | Biz Spec §6.3 |
| 2차 가설 | \[%\] | \[타겟\] | \[행동\] | \[Go/Kill 경계선\] | Biz Spec §6.3 |

**좋은 XYZ 가설 체크:**

* X%는 구체적 숫자 (예: 15%, 30%). "많은" "대부분" 금지.  
* Y는 좁게 정의 ("30대 직장인" ✗ → "주 3회 이상 두통약 복용하는 30대 직장인" ✓)  
* Z는 관찰 가능한 행동 ("관심을 보인다" ✗ → "이메일 주소를 남긴다" ✓)  
* 성공 기준은 Go/Kill을 가르는 경계선. 모호하면 판정이 흐려진다.

### **1-2. Kill Assumptions 연결**

Business Spec §7.1의 핵심 가정(Kill Assumptions) 중 이 실험으로 검증 가능한 것. Kill Assumption이 틀리면 사업 자체가 성립하지 않는다. 이 실험이 그것을 확인하는 기회다.

| \# | Kill Assumption (Biz Spec §7.1) | 이 실험과의 관계 | 검증 가능 여부 |
| ----- | ----- | ----- | ----- |
| 1 | \[핵심 가정\] | \[어떻게 검증되는가\] | Y/N |
| 2 | \[핵심 가정\] | \[어떻게 검증되는가\] | Y/N |

## **§5. Go/Pivot/Kill 판정**

이 Gate가 Kill Funnel에서 가장 중요하다. 여기서 Kill되면 프로세스 종료. 판정은 데이터로 한다. "느낌"으로 하지 않는다.

### **5-1. 판정 스코어카드**

| 기준 | Go | Pivot | Kill | 우리 결과 |
| ----- | ----- | ----- | ----- | ----- |
| **XYZ 가설** | 목표치 달성 | 부분 달성 (방향 O, 수치 미달) | 달성률 현저히 낮음 | \[결과\] |
| **Skin in the Game** | 30점+ 행동 데이터 | 10점 수준 (관심 O, 행동 약) | 0점 (의견만) | \[\_\_\_점\] |
| **NPQ** | 3가지 중 2개+ 충족 | 1개 충족 | 0개 | \[\_\_\_/3 충족\] |
| **시장 신호** | 명확한 수요 신호 | 모호 → 가설 수정 후 재실험 | 수요 부재 | \[결과\] |

### **5-2. 판정 결과**

* **판정:** \[Go / Pivot / Kill\]  
* **핵심 근거:** \[1\~2문장. 데이터 기반.\]  
* **판정자:** \[이름\] (날짜: YYYY-MM-DD)

**임원회의 안건:** 이 문서 \+ 데이터 → Go/Pivot/Kill 최종 결정.

### **5-3. Pivot 시 (해당 시에만 작성)**

| 항목 | 내용 |
| ----- | ----- |
| **Pivot 방향** | \[무엇을 바꾸는가\] |
| **Brief 수정 사항** | \[Product Brief 어느 섹션을 어떻게 수정\] |
| **재실험 계획** | \[Stage 2를 1회 더 반복. 변경점 \+ 기간\] |

Pivot은 1회만 허용. 2번째 Pivot이면 Kill을 심각하게 고려한다.

### **5-4. Go 시 \- Stage 3 인풋 요약**

Product Spec 착수를 위해, 실험에서 확인된 핵심 사실을 압축한다. 이 섹션이 Stage 3의 시작점이 된다.

| 항목 | 내용 |
| ----- | ----- |
| **검증된 핵심 가치** | \[실험으로 확인된 고객이 원하는 것\] |
| **핵심 사용자 행동** | \[가장 강한 신호를 보인 행동\] |
| **P0 기능 시사점** | \[실험 결과가 Brief §6 스코프에 주는 시사점\] |
| **주의할 리스크** | \[실험에서 발견된 새로운 리스크/우려\] |

## **작성 가이드**

### **PM → AI 워크플로우**

| Step | 누가 | 시간 | 행동 |
| ----- | ----- | ----- | ----- |
| 1 | PM | 30분 | Biz Spec §3, §6.3에서 XYZ 가설·NPQ·Skin in the Game 추출. 실험 방향 결정. |
| 2 | AI | 20분 | 실험 설계 초안 (기법 추천, 측정 계획, 타임라인, 표본 크기 산출) |
| 3 | PM | 10분 | 실험 설계 검토·확정. §1\~§2 완성. |
| 4 | PM | 2주 | 현장 실험 실행. AI가 대체 불가. PM이 직접 고객 접점에서 수행. |
| 5 | AI | 30분 | 결과 데이터 정리 \+ NPQ 재판정 \+ 스코어카드 초안 |
| 6 | PM | 30분 | 판정 최종 결정 \+ 근거 작성. Go 시 §5.4 작성. |

**핵심:** Step 4(현장 실험)는 AI가 대체할 수 없다. 실험 설계와 데이터 분석은 AI가 주도하되, 고객 접촉은 반드시 사람이 한다.
