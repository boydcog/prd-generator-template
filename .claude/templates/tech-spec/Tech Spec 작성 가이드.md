# **Tech Spec 작성 가이드 v2.0**

LLM이 최적 성능으로 개발하도록 최소한의 가드레일만 제공하는 기술 사양서

* **핵심 철학:** "LLM에게 최소 제약을 준다. 구조가 적을수록 LLM이 더 잘 판단한다."
* **프로세스 위치:** Stage 3 초안 → Stage 4 Freeze 완성·확정
* **Gate 연결:** Gate 4 (인계 패키지 완성 — Design Spec + Tech Spec + Product Spec 전원 합의)
* **작성:** 엔지니어 주도 (PM · 디자이너 3자 리뷰 후 확정)
* **데이터 수준:** 확정 (Core Stack 최종 결정 + 외부 연동 확정 기반)

---

## **기존 방식과의 차이**

| 기존 Tech Spec (§1\~§8) | 새 Tech Spec (항목 1\~4) |
| ----- | ----- |
| API 명세 상세 기술 | API는 LLM이 Product Spec AC 보고 자율 설계 |
| 데이터 모델 ER 다이어그램 | 표준 원칙만 제시 (항목 2), 구체 모델은 LLM 판단 |
| 인프라·배포 섹션 | Core Stack 선택으로 대체 (인프라는 Stack에서 결정) |
| 보안·성능 상세 기준 | 표준 구현 규칙(항목 3)으로 커버 |
| 8개 섹션, 20\~30분 작성 | **4개 항목, 10\~15분 작성** |

> **왜 줄였는가?** 상세 명세를 미리 작성하면 LLM이 명세를 따르는 데 집중하여 창의적 최적 해법을 찾지 못함. 가드레일(변해선 안 되는 것)만 명시하고, 나머지는 LLM에게 위임.

## **각 항목 상세 설명**

### **항목 1 — Core Stack**

후보군에서 **프로젝트 성격에 맞게 선택**한다. 선택 기준:

| 질문 | 선택 기준 |
| ----- | ----- |
| 모바일 앱인가? | React Native + Expo 선택 |
| 웹앱인가? | Next.js App Router 선택 (SPA가 필요한 경우만 Vite) |
| BLE/하드웨어 연동 있는가? | Expo SDK (BLE 플러그인 지원) |
| 실시간 기능 있는가? | Supabase (Realtime 지원) |
| 간단한 백엔드인가? | Supabase Edge Functions (서버 불필요) |
| AI/LLM 필수인가? | Anthropic Claude API 우선 (이모코그 표준) |

모든 레이어를 선택한 후 **버전까지 명시**한다.

---

### **항목 2\~3 — 이모코그 표준 원칙**

이모코그 전 제품에 공통 적용되는 표준이다. **변경 불필요.**

추가가 필요한 경우 항목 2·3에 **행을 추가**하되, 기존 행은 삭제하지 않는다.

예시 추가:
- 항목 2에 추가: "6. BLE 센서 데이터는 raw GATT bytes로 저장 후 앱에서 파싱"
- 항목 3에 추가: "8. React Native에서 `alert()`·`console.log()` 프로덕션 잔류 금지"

---

### **항목 4 — Key Integration**

이 제품에 필요한 **외부 연동만** 작성한다. 불필요한 섹션(External Device, Payment 등)은 삭제한다.

| 연동 유형 | 작성 기준 |
| ----- | ----- |
| **Auth** | 모든 제품 필수. Provider + 인증 방식 명시 |
| **Storage** | 파일/미디어 업로드가 있는 제품 |
| **External Device** | BLE 센서, 웨어러블, 의료기기 연동이 있는 제품 |
| **Payment** | 유료 기능, 구독, In-App Purchase가 있는 제품 |

> Product Spec §7-3 외부 연동 목록과 1:1 대응 확인 필수.

---

### **항목 5 — API Design Convention (선택)**

이 항목은 **선택적으로 추가**한다. 다음 경우 추가 권장:

- REST API 엔드포인트가 10개 이상
- 여러 프론트엔드 클라이언트가 동일 API를 공유
- 외부 파트너에게 API를 제공하는 경우

추가 형식:

```markdown
## 5. API Design Convention

| Method | Endpoint | 설명 |
| ----- | ----- | ----- |
| GET | /api/[리소스] | [설명] |
| POST | /api/[리소스] | [설명] |
| PUT | /api/[리소스]/[id] | [설명] |
| DELETE | /api/[리소스]/[id] | [설명] |
```
