# Tech Spec 작성 가이드 v2.0

LLM이 최적 성능으로 개발하도록 최소한의 가드레일만 제공하는 기술 사양서

- **핵심 철학:** 상세한 API 명세나 데이터 모델을 작성하지 않는다. LLM이 스스로 판단한다. 이 문서는 LLM이 어겨선 안 되는 최소 제약만 정의한다.
- **프로세스 위치:** S4 Freeze 단계 핸드오프 문서
- **Gate 연결:** Gate 4 — PM·디자이너·엔지니어 3자 합의 후 S5 개발 진입
- **작성자:** 엔지니어 주도, PM 검토

## 각 항목 작성 상세

### 항목 1 (Core Stack): 후보군에서 프로젝트 성격에 맞게 선택

레이어별 후보군에서 하나씩 선택하고 버전을 명시한다. 해당 없는 레이어는 "해당없음"으로 표시한다.

선택 기준 예시:
- 모바일 앱 → Mobile에서 React Native + Expo 선택, Frontend는 해당없음
- 웹 서비스 → Mobile은 해당없음, Frontend에서 Next.js 선택
- BLE 디바이스 연동 → Mobile 필수, External Device 섹션 작성

### 항목 2~3: 이모코그 표준 원칙

변경 불필요. 추가가 필요한 경우 항목 끝에 bullet을 추가한다. 기존 항목 삭제 불가.

### 항목 4 (Key Integration): 제품별 필수 작성

Product Spec §7-3 EXT 목록을 기반으로 작성한다. 해당 없는 섹션(Storage, Payment, External Device 등)은 통째로 삭제한다.

### 항목 5 (API Design Convention): 선택 섹션

다음 경우에만 추가한다:
- 프론트엔드-백엔드 간 API 계약이 복잡한 경우
- 여러 엔지니어가 동시에 작업하는 경우
- 외부 팀과 API를 공유하는 경우

단순한 CRUD 앱이라면 LLM이 알아서 설계하도록 두고 이 섹션은 작성하지 않는다.
