## **2\. 최소 연동 구조**

전체 아키텍처를 상세히 서술하는 것이 아닌, **AI 코딩 도구와 엔지니어가 알아야 할 핵심 연동 포인트**만 기술한다.

### **2-1. 레이어 책임 요약**

| 레이어 | 책임 | 포함되지 않는 것 |
| ----- | ----- | ----- |
| **UI 레이어** | 화면 렌더링, 사용자 입력 | 비즈니스 로직 |
| **API 레이어** | 요청 검증, 응답 형식화 | 직접 DB 접근 |
| **비즈니스 로직** | 도메인 규칙, 워크플로우 | UI 의존성 |
| **데이터 레이어** | DB CRUD, 트랜잭션 | 비즈니스 규칙 |
| **외부 서비스 레이어** | Product Spec §7-3 연동 목록 처리 | UI 로직 |

### **2-2. 핵심 데이터 흐름**

가장 중요한 P0 기능의 흐름만 기술한다. 세부 구현은 §3~§4에서.

```
[시나리오: 핵심 기능 실행]

1. 사용자 → [화면/SCR-N]에서 [액션]
2. [컴포넌트] → POST /api/[endpoint]
3. API Route → zod 검증
4. API Route → [비즈니스 서비스].execute()
5. 서비스 → [데이터 레이어].query()
6. (필요시) 서비스 → 외부 API 호출 (EXT-00N)
7. 결과 → JSON 응답 → UI 업데이트
```

### **2-3. 외부 연동 연결 요약**

Product Spec §7-3 연동 목록과 내부 레이어를 매핑한다.

| 연동 ID | 서비스명 | 호출 위치 (레이어) | 인증 방식 | 실패 시 폴백 |
| ----- | ----- | ----- | ----- | ----- |
| EXT-001 | \[서비스명\] | \[외부 서비스 레이어\] | \[API Key / OAuth\] | \[폴백 동작\] |
| EXT-002 | \[서비스명\] | \[레이어\] | \[인증 방식\] | \[폴백\] |

## **3\. API 명세**

**우선순위:** Product Spec §7-3 외부 연동 목록의 EXT-00N 항목을 우선 명세한다. 내부 API는 그 다음.

### **3-1. API 엔드포인트 목록**

| Method | Path | 설명 | 인증 | 관련 US | 연동 EXT |
| ----- | ----- | ----- | ----- | ----- | ----- |
| GET | /api/\[리소스\] | \[설명\] | \[Yes/No\] | US-001 | — |
| POST | /api/\[리소스\] | \[설명\] | \[Yes/No\] | US-002 | EXT-001 |
| PUT | /api/\[리소스\]/\[id\] | \[설명\] | \[Yes/No\] | US-003 | — |
| DELETE | /api/\[리소스\]/\[id\] | \[설명\] | \[Yes/No\] | US-004 | — |

> **Product Spec §7-3 매핑:** 위 테이블의 `연동 EXT` 열이 Product Spec §7-3의 EXT-00N과 1:1 대응되어야 한다.

### **3-2. 주요 엔드포인트 상세**

#### **POST /api/\[리소스\]**

**목적:** \[이 API가 하는 일\]

**요청**

```typescript
{
  field1: string,
  field2: number,
  field3?: string,
}
```

**응답 (성공 200)**

```typescript
{
  id: string,
  field1: string,
  createdAt: string,
}
```

**에러 응답**

```typescript
{
  error: {
    code: string,
    message: string,
  }
}
```

**에러 코드**

| HTTP | 에러 코드 | 의미 |
| ----- | ----- | ----- |
| 400 | VALIDATION\_ERROR | 입력값 오류 |
| 401 | UNAUTHORIZED | 인증 필요 |
| 404 | NOT\_FOUND | 리소스 없음 |
| 500 | INTERNAL\_ERROR | 서버 오류 |

## **5\. AI Rules & 가드레일**

### **5-1. 프로젝트 전역 Rules**

`.cursor/rules/project.md` 또는 `CLAUDE.md`에 저장.

```markdown
# [프로젝트명] AI Rules

## 아키텍처
- [규칙 1: 예) 모든 DB 접근은 /lib/db/ 레이어를 통해서만]
- [규칙 2: 예) API Routes는 비즈니스 로직을 포함하지 않는다]
- [규칙 3: 예) 서버 컴포넌트를 기본으로, 인터랙션이 필요한 경우만 "use client"]

## 코딩 컨벤션
- TypeScript strict 모드. any 타입 금지.
- 파일명: kebab-case. 컴포넌트: PascalCase.
- [기타 컨벤션]

## 금지 사항
- [예: 환경 변수 하드코딩 금지]
- [예: console.log 프로덕션 코드 잔류 금지]

## 보안
- [예: 사용자 입력은 항상 zod로 검증]
- [예: SQL은 ORM을 통해서만 실행]
```
