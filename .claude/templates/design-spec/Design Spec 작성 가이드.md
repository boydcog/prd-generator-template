# **Design Spec 작성 가이드 v2.0**

시안이 아닌 규칙. AI가 이 규칙만으로 디자인 시스템을 준수하는 UI를 생성한다.

* **핵심 질문:** "AI가 이 규칙만으로 디자인 시스템을 준수하는 UI를 생성할 수 있는가?"
* **프로세스 위치:** Stage 4 — Freeze (0.5주)
* **Gate 연결:** Gate 4 (인계 패키지 완성 — 4종 문서 전원 합의)
* **작성:** 디자이너 (Product Spec §3,§4 기반 분리·확장. PM·엔지니어 3자 리뷰 후 확정)
* **데이터 수준:** 확정 (Prototype 결과 + 사용성 테스트 반영)

## **Zero-Design-Handoff 원칙**

이 문서는 "시안"이 아니라 "규칙"이다.

| 기존 방식 | Zero-Design-Handoff |
| ----- | ----- |
| 디자이너 → 50개 시안 (2-3주) → 개발자 | PM → Spec(디자인 규칙) → AI 생성 → 디자이너 QA |
| Figma 파일 전달 | 규칙 문서 전달 |
| 화면별 ASCII 와이어프레임 | 실제 콘텐츠 명세 + 비주얼 시스템 |
| 디자이너가 모든 화면 제작 | AI가 규칙 기반으로 생성, 디자이너가 QA |

**Track A (기본 — 이모코그 표준):** shadcn/ui + Tailwind + AI 코딩 도구
**Track B (예외 — 브랜드 강도 높을 때):** Figma 시안 + 이 문서를 Figma 연동 가이드로 활용

## **4. 컴포넌트 고도화**

이 섹션이 답하는 질문: "어떤 컴포넌트를 어떻게 쓰고, 상태는 어떻게 표현하는가? 마이크로인터랙션과 접근성 기준은?"
**기본 라이브러리:** shadcn/ui (Track A). 커스텀이 필요한 경우만 별도 정의.

### **4-1. 컴포넌트 사용 규칙**

| UI 요소 | shadcn 컴포넌트 | 커스텀 여부 | 사용 규칙 |
| ----- | ----- | ----- | ----- |
| 기본 버튼 | Button (variant="default") | 표준 | \[언제 쓰는가\] |
| 보조 버튼 | Button (variant="outline") | 표준 | \[언제 쓰는가\] |
| 텍스트 입력 | Input | 표준 | \[placeholder 규칙\] |
| 드롭다운 | Select | 표준 | \[선택지 3개 초과 시\] |
| 모달 | Dialog | 표준 | \[확인이 필요한 액션에만\] |
| 알림 | Toast | 표준 | \[성공/에러 피드백\] |
| \[커스텀 요소\] | — | 커스텀 | \[왜 커스텀이 필요한가\] |

### **4-2. 상태별 비주얼 명세**

| 상태 | 시각 표현 | 색상/투명도 | 적용 컴포넌트 |
| ----- | ----- | ----- | ----- |
| **Default** | \[기본 상태\] | — | \[컴포넌트\] |
| **Hover** | \[hover 시 색상 강도 변화\] | \[Tailwind 클래스\] | \[컴포넌트\] |
| **Active/Pressed** | \[누른 상태 피드백\] | \[클래스\] | \[컴포넌트\] |
| **Focus** | \[포커스 링 색상/두께\] | `ring-2 ring-primary` | 인풋, 버튼 |
| **Loading** | Skeleton / Spinner — \[어느 것을 언제\] | opacity-70 | \[컴포넌트 목록\] |
| **Empty** | \[빈 상태 메시지 + CTA\] | muted-foreground | \[컴포넌트 목록\] |
| **Error** | \[에러 색상(destructive) + 메시지 + 복구 액션\] | destructive | \[컴포넌트 목록\] |
| **Disabled** | opacity-50 + cursor-not-allowed | opacity-50 | \[컴포넌트 목록\] |
| **Success** | \[토스트 + 색상 피드백\] | \[green 계열\] | Toast |

### **4-3. 마이크로인터랙션 규칙**

| 패턴 | 트리거 | 피드백 | 지속 시간 | 예외 |
| ----- | ----- | ----- | ----- | ----- |
| **버튼 클릭 피드백** | 클릭 | 즉시 Loading. 응답 대기 중 재클릭 차단 | 응답까지 | \[예외\] |
| **폼 유효성 검사** | Submit + 포커스 아웃 | 필드별 에러 메시지 | 수정 전까지 | \[예외\] |
| **토스트 알림** | 성공/에러 발생 | Toast 팝업 | 3초 후 자동 소멸 | \[예외\] |
| **페이지 전환** | 링크/버튼 클릭 | 즉시 전환 (느린 경우만 Skeleton) | — | \[예외\] |
| **모달 닫기** | ESC + 배경 클릭 + X버튼 | 즉시 닫힘 | — | 파괴적 액션은 배경클릭 차단 |
| **에러 복구** | 에러 발생 | 에러 메시지 + 구체적 복구 방법 | 수정 전까지 | \[이유\] |

### **4-4. 접근성 기준 (WCAG 2.1 AA)**

| 항목 | 기준 | 구현 방법 |
| ----- | ----- | ----- |
| **색상 대비** | 텍스트 4.5:1 이상 (AA 레벨) | Contrast Checker 검증 |
| **버튼 aria-label** | 텍스트 없는 아이콘 버튼에 필수 | `aria-label="[설명]"` |
| **이미지 alt** | 의미 있는 이미지 전부 | `alt="[설명]"`, 장식용 이미지 `alt=""` |
| **키보드 네비게이션** | Tab 순서 논리적, 포커스 가시적 | `focus-visible:ring-2` |
| **스크린리더 테스트** | 핵심 P0 기능 전부 | \[테스트 도구\] |
| **터치 타겟** | 최소 44×44px | Tailwind: `min-h-11 min-w-11` |

## **6. AI 생성 지침**

이 섹션이 답하는 질문: "AI 코딩 도구에게 무엇을 어떻게 지시하는가?"
**이 섹션이 Zero-Design-Handoff의 핵심.** AI(Cursor, Lovable, v0 등)가 이 규칙을 읽고 UI를 생성한다.

### **6-1. 기술 스택 지침**

```
[AI에게 주는 지침 - 그대로 복사하여 사용]

기술 스택:
- Framework: Next.js 14 (App Router)
- UI: shadcn/ui + Tailwind CSS
- 아이콘: lucide-react
- 폼: react-hook-form + zod
- 상태: [React Context / Zustand / 기타]
- 이미지: next/image (WebP, lazy loading 자동 적용)

컴포넌트 규칙:
- shadcn/ui 컴포넌트를 최우선으로 사용한다.
- 커스텀 컴포넌트가 필요한 경우만 새로 만든다.
- className에는 Tailwind 유틸리티 클래스만 사용한다.
- CSS 파일을 새로 만들지 않는다.

접근성:
- 텍스트 없는 버튼에는 aria-label 필수.
- 이미지에 alt 속성 필수 (장식용은 alt="").
- 색상 대비 WCAG AA(4.5:1) 이상 유지.
```

### **6-2. 비주얼 시스템 지침**

```
[AI에게 주는 지침]

디자인 토큰 (globals.css의 CSS 변수 사용):
- Primary: [색상값] → Tailwind: bg-primary, text-primary
- Background: [색상값] → Tailwind: bg-background
- Success: [색상값] → Tailwind: text-green-600
- Warning: [색상값] → Tailwind: text-amber-500

타이포그래피:
- 본문: font-sans text-base text-foreground leading-relaxed
- 제목: font-bold text-2xl tracking-tight
- 보조: text-sm text-muted-foreground

레이아웃:
- 페이지 패딩: px-4 (모바일) / px-6 (sm 이상)
- 컨테이너: max-w-[값] mx-auto
- 섹션 간격: space-y-6 또는 gap-6

이미지:
- next/image 컴포넌트 사용. width, height 또는 fill 지정.
- placeholder="blur" 적용.
- 에셋 소스: [CDN URL 패턴]
```
