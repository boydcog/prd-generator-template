# **Design Spec 작성 가이드 v2.0**

AI가 이모코그 표준 디자인 시스템을 준수하는 UI를 생성하기 위한 규칙 문서

* **핵심 질문:** "AI 코딩 도구가 이 규칙만으로 이모코그 스타일 일관성을 유지하는 UI를 생성할 수 있는가?"
* **프로세스 위치:** Stage 4 — Freeze (0.5주)
* **Gate 연결:** Gate 4 (인계 패키지 완성 — Design Spec + Tech Spec + Product Spec 전원 합의)
* **작성:** 디자이너 주도 (Product Spec §7 기반, PM·엔지니어 3자 리뷰 후 확정)
* **데이터 수준:** 확정 (Prototype 결과 + 타겟 사용자 인사이트 반영)

---

## **Zero-Design-Handoff 원칙 v2**

이 문서는 **"화면 시안"이 아닌 "AI가 읽는 디자인 규칙"**이다.

| 기존 방식 | Zero-Design-Handoff v2 |
| ----- | ----- |
| 디자이너 → 50개 Figma 시안 → 개발자 | 규칙 문서(Design Spec) → AI 생성 → 디자이너 QA |
| 화면별 상세 명세 | 이모코그 표준 + 제품별 조정 포인트 |
| shadcn/ui 범용 | gluestack-ui v2 + Tailwind v4 + Framer Motion |
| 매 제품 처음부터 작성 | 항목 1\~5 prefill, 항목 6\~9 조정, 항목 10\~11만 신규 작성 |

## **각 항목 상세 설명**

### **항목 1\~5 — 이모코그 표준**

이 항목들은 **이모코그 기술 스택(gluestack-ui v2 + Tailwind v4 + Framer Motion)과 디자인 원칙**을 반영한 표준이다.
**변경 금지.** 변경이 필요한 특수 사유가 있다면 Tech Spec 담당자와 협의 후 Design Spec에 명시한다.

> 예: "이 제품은 gluestack-ui 미사용 — 이유: React Native Web 환경 아님, Expo SDK 제약"

**항목 1 (기술 스택):** React + gluestack-ui v2 + Tailwind v4 + Framer Motion 고정.
**항목 2 (토큰 아키텍처):** Primitive → Semantic → Component Token 계층 고정. 서비스별 추가 규칙만 하단에 작성.
**항목 3 (컴포넌트):** gluestack-ui v2 기본 컴포넌트 우선. Size 정책만 타겟에 따라 선택(sm/md/lg 또는 md/lg/xl).
**항목 4 (타이포그래피):** Pretendard 폰트 + Tailwind 타입 스케일 고정. 최소 크기 규칙만 조정.
**항목 5 (아이콘):** lucide-react 고정. 크기/색상 규칙 불변.

---

### **항목 6\~8 — 기본값 제공, 타겟에 따라 조정**

기본값이 prefill되어 있으나, **타겟 사용자 특성에 따라 조정 가능하다.**
조정 시 `[변경 포인트]` 주석이 있는 값만 변경한다.

**항목 6 (인터랙션):**
- 일반 타겟: duration 0.3s/0.4s
- 시니어/저연령 타겟: duration 0.4s/0.5s (느리고 명확한 피드백)
- 선택 후 해당 값을 §9 CSS 변수 또는 코드 주석으로 명시

**항목 7 (접근성):**
- 일반 타겟: WCAG AA (4.5:1 텍스트 대비)
- 시니어/저시력 타겟: WCAG AAA (7:1 대비) + Touch Guard (최소 64px 터치 타겟)
- WCAG 레벨 선택 → §9의 `--foreground` 값 조정으로 연결

**항목 8 (레이아웃):**
- Mobile 섹션은 모든 제품에 필수
- Web/Dashboard 레이아웃은 해당 뷰가 있는 경우에만 작성. 없으면 섹션 삭제.

---

### **항목 9 — CSS 변수**

`globals.css`의 Tailwind v4 CSS 변수 정의. **`[변경 포인트]` 주석이 달린 값만 수정한다.**

주요 조정 대상:
- `--primary`: 브랜드 컬러 (OKLCH hue 값으로 교체)
- `--foreground`: 기본 `0.208`. 고대비 필요 시 `0.15`로 낮춤
- 다크모드 불필요 제품: `.dark` 블록 전체 삭제

---

### **항목 10\~11 — 완전 신규 작성**

이 두 항목은 **이 제품에만 해당하는 내용**으로, prefill이 없다. 반드시 작성해야 한다.

**항목 10 (타겟 맞춤 UX 전략):**
Product Spec §7의 타겟 사용자 인사이트를 기반으로 작성.
AI에게 "이 제품 사용자는 어떤 맥락에서, 어떤 UX 장벽을 겪는가"를 전달하는 섹션.

**항목 11 (최종 품질 검수 항목):**
P0 기능 기반 QA 체크리스트. AI 생성 UI 결과물을 검수하는 기준.
타겟 사용자가 실수하기 쉬운 인터랙션, 접근성 기준 확인 방법 포함.
