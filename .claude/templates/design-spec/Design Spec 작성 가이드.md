# Design Spec 작성 가이드 v2.0

AI 코딩 도구가 이모코그 표준 디자인 시스템을 준수하는 UI를 생성하기 위한 규칙 문서

- **핵심 질문:** "AI 코딩 도구가 이 규칙만으로 이모코그 스타일 일관성을 유지하는 UI를 생성할 수 있는가?"
- **프로세스 위치:** S4 Freeze 단계 핸드오프 문서
- **Gate 연결:** Gate 4 — PM·디자이너·엔지니어 3자 합의 후 S5 개발 진입
- **작성자:** 디자이너 주도, PM 검토, 엔지니어 확인
- **데이터 수준:** 항목 1~9 이모코그 표준 prefill + 서비스별 조정. 항목 10~11 신규 작성

## 각 항목 작성 상세

### 항목 1~5: 이모코그 표준

변경 금지. 이 섹션들은 emocog 전체 제품군의 일관성을 위한 최소 공통 분모이다. 변경이 필요한 경우 반드시 Tech Spec 담당자와 협의 후 사유를 명시한다.

### 항목 6~8: 기본값 제공, 타겟 사용자에 따라 조정 가능

각 항목의 **[조정 포인트]** 주석이 달린 부분을 제품 타겟에 맞게 수정한다.
- 일반 성인 타겟: 기본값 그대로 사용
- 시니어/저시력 타겟: 주석에 표시된 조정값 적용

### 항목 9: CSS 변수 정의

코드블록 전체가 prefill됨. **[변경 포인트]** 주석이 달린 값만 수정한다.
- `--foreground`: 기본 `0.208` → 고대비 필요 시 `0.15`로 낮춤
- `--color-primary-500`: 브랜드 컬러로 교체 (OKLCH hue: 250=파랑, 145=초록, 25=빨강)
- 다크모드 불필요 시 `.dark` 블록 전체 삭제

### 항목 10~11: 완전 신규 작성

Product Spec을 읽고 이 제품 전용으로 작성한다. 예시 파일(`기억생생 Design Spec 예시.md`)을 참고하되 그대로 복사하지 않는다.

**항목 10 작성 프롬프트:**
- 이 제품의 타겟 사용자는 어떤 맥락에서 사용하는가? (장소, 시간, 기기 상태)
- 사용자가 가장 자주 겪는 UX 장벽은 무엇인가?
- 이 장벽을 해소하기 위해 이 제품이 채택하는 UX 전략은?

**항목 11 작성 프롬프트:**
- P0 기능에서 반드시 통과해야 하는 UX 시나리오는?
- 타겟 사용자가 실수하기 쉬운 인터랙션은?
- 항목 7 접근성 기준이 실제 UI에 적용되었는지 어떻게 검증하는가?
