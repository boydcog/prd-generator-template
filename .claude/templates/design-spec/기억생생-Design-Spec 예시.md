# **기억생생 Design Spec**

버전: v1.0 | 업데이트: 2026-03-04 | 작성자: emocog design team

> **이 문서는 기억생생(시니어 기억 훈련 앱)의 Design Spec 예시입니다.**
> 항목 1\~5는 이모코그 표준 그대로. 항목 6\~9는 시니어 타겟 기준으로 조정됨.
> 항목 10\~11은 시니어 기억 훈련 앱 전용 내용으로 완성됨.

## **2\. 토큰 기반 아키텍처**

> 이모코그 표준 + 시니어 타겟 추가 규칙

### **2-1. 토큰 계층 구조**

```
Primitive Tokens (CSS 변수, §9 참조)
  └── Semantic Tokens (bg-background, text-foreground 등)
        └── Component Tokens (gluestack-ui 컴포넌트 props)
              └── Page Composition
```

### **2-2. 토큰 사용 규칙**

| 규칙 | 내용 |
| ----- | ----- |
| **색상** | 항상 semantic token 사용. `#hex` 직접 사용 금지 |
| **간격** | Tailwind 4px 단위 사용 |
| **오클릭 방지** | 인터랙티브 요소 간 최소 `gap-3` (12px) — 시니어 타겟 기준 |
| **폰트** | semantic class 사용 |
| **radius** | `rounded-lg`(12px) 이상 사용 — 시니어 친화적 둥근 모서리 |

### **2-3. 서비스별 추가 토큰 규칙**

- 훈련 성공 시 축하 애니메이션: `--success` 토큰으로 배경 펄스 효과
- 훈련 실패 시: 빨간색 직접 사용 금지, `--destructive` 토큰 경유
- 점수/레벨 표시: `--primary` 강조 색상 사용

## **3\. 컴포넌트 라이브러리**

### **3-1. 기본 컴포넌트 사용 규칙**

| UI 요소 | gluestack 컴포넌트 | Tailwind 스타일링 | 사용 규칙 |
| ----- | ----- | ----- | ----- |
| **버튼 (Primary)** | `<Button>` | `bg-primary text-primary-foreground` | P0 CTA, 훈련 시작/완료 |
| **버튼 (Secondary)** | `<Button>` | `bg-secondary text-secondary-foreground` | 설정, 보조 액션 |
| **카드 (훈련)** | `<Card>` | `bg-card rounded-xl shadow-md` | 기억 카드 — 큰 터치 영역 |
| **모달** | `<Modal>` | `bg-background rounded-2xl` | 결과 확인, 하루 목표 달성 |
| **토스트** | `<Toast>` | 상태별 색상 | 정답/오답 즉시 피드백 |
| **로딩** | Skeleton | `bg-muted animate-pulse rounded-xl` | 훈련 콘텐츠 로딩 |

### **3-2. 컴포넌트 Size 정책**

> **기억생생: md·lg·xl 사용. sm 사용 금지 (시니어 오클릭 방지)**

| Size | 버튼 높이 | 패딩 | 텍스트 | 적용 대상 |
| ----- | ----- | ----- | ----- | ----- |
| **sm** | 32px | — | — | **사용 안 함** |
| **md** | 40px | px-4 py-2 | text-base | 설정, 보조 버튼 |
| **lg** | 48px | px-6 py-3 | text-lg | 네비게이션 버튼 |
| **xl** | 56px | px-8 py-4 | text-xl | **주요 CTA** (훈련 시작, 정답 선택) |

### **3-3. 상태별 비주얼**

| 상태 | 시각 표현 | Tailwind 클래스 |
| ----- | ----- | ----- |
| **Default** | 기본 | — |
| **Hover** | 배경 10% 어둡게 | `hover:opacity-90` |
| **Active/Pressed** | 배경 20% 어둡게 + 크기 살짝 줄어듦 | `active:opacity-80 active:scale-[0.97]` |
| **Focus** | 굵은 포커스 링 (시니어 선명성) | `focus-visible:ring-4 focus-visible:ring-ring` |
| **Loading** | Skeleton | `opacity-70` |
| **Disabled** | 반투명 | `opacity-50 cursor-not-allowed` |
| **Correct (정답)** | 초록 배경 + 체크 아이콘 | `bg-success text-success-foreground` |
| **Incorrect (오답)** | 연한 배경 (빨간색 과도 사용 금지) | `bg-muted border-destructive` |

## **6\. 인터랙션**

> **기억생생: 시니어 타겟 조정값 적용 (느리고 명확한 피드백)**

### **6-1. Framer Motion 트랜지션**

| 트랜지션 | 기억생생 설정 | 적용 대상 |
| ----- | ----- | ----- |
| **빠른 피드백** | duration: 0.2s, ease: \[0.4,0,0.2,1\] | 버튼 hover/press |
| **화면 전환** | duration: 0.4s, ease: \[0.4,0,0.2,1\] | 페이지/모달 진입 |
| **콘텐츠 표시** | duration: 0.5s, ease: \[0,0,0.2,1\] | 기억 카드, 훈련 결과 |
| **종료** | duration: 0.3s, ease: \[0.4,0,1,1\] | 모달 닫기 |
| **정답 축하** | duration: 0.6s, spring (stiffness: 200, damping: 15) | 정답 카드 바운스 |

### **6-2. 마이크로인터랙션 규칙**

| 패턴 | 트리거 | 피드백 | 지속 시간 |
| ----- | ----- | ----- | ----- |
| **정답 선택** | 카드 탭 | 초록 배경 + 체크 아이콘 + 경쾌한 바운스 | 0.8초 후 다음 카드 |
| **오답 선택** | 카드 탭 | 테두리 강조 (과도한 빨간색 금지) + 진동 | 1.2초 대기 후 재시도 |
| **훈련 완료** | 마지막 문제 | 별 3개 애니메이션 + "잘 하셨어요!" | 2초 → 결과 화면 |
| **시간 초과** | 타이머 종료 | 차분한 알림 (급박한 경고 금지) | 즉시 다음으로 |
| **세션 시작** | 시작 버튼 | 카운트다운 3→2→1 (천천히) | 3초 |

## **7\. 접근성 및 가독성**

> **기억생생: WCAG AAA 기준 + Touch Guard 적용**

| 항목 | 기억생생 기준 | 구현 방법 |
| ----- | ----- | ----- |
| **텍스트 대비** | WCAG AAA 7:1 | CSS 변수 §9의 `--foreground: oklch(0.15 0 0)` |
| **UI 컴포넌트 대비** | 4.5:1 | 버튼 테두리, 카드 테두리 강화 |
| **터치 타겟** | 64×64px (`min-h-16 min-w-16`) | 훈련 카드, 주요 버튼 |
| **포커스 링** | `ring-4 ring-ring` | focus-visible (시니어 선명성) |
| **텍스트 최소 크기** | `text-base` (16px) 이상 | §4-3 참조 |
| **Touch Guard** | 인터랙티브 요소 간 `gap-3` 이상 | 오클릭 방지 |
| **스크린리더** | 전체 화면 | aria-label, role, aria-live |
| **색상 의존 금지** | 정답/오답 표시 시 아이콘 병행 | ✓ 체크, ✗ X 아이콘 항상 함께 |

**적용 기준:** WCAG AAA 기준 적용 — 시니어(60세 이상) 기억 훈련 앱

## **9\. CSS 변수 정의**

> **기억생생: 고대비 포레그라운드 + 따뜻한 브랜드 컬러 (보라-파랑 계열)**

```css
/* globals.css — 기억생생 디자인 토큰 */
@layer base {
  :root {
    /* ─── Background / Surface ─── */
    --background: oklch(1 0 0);                  /* 흰 배경 */
    --card: oklch(0.98 0.01 280);                /* 약한 보라 틴트 카드 */
    --popover: oklch(1 0 0);

    /* ─── Foreground / Text ─── */
    --foreground: oklch(0.15 0 0);              /* 고대비 (WCAG AAA용, 기본 0.208 → 0.15로 낮춤) */
    --card-foreground: oklch(0.15 0 0);
    --muted: oklch(0.95 0.01 280);
    --muted-foreground: oklch(0.45 0 0);

    /* ─── Brand / Primary (기억생생: 보라-파랑) ─── */
    --primary: oklch(0.52 0.20 280);            /* 브랜드 보라 */
    --primary-foreground: oklch(1 0 0);
    --secondary: oklch(0.90 0.03 280);
    --secondary-foreground: oklch(0.20 0 0);

    /* ─── Feedback ─── */
    --destructive: oklch(0.55 0.22 25);
    --destructive-foreground: oklch(1 0 0);
    --success: oklch(0.52 0.16 145);
    --success-foreground: oklch(1 0 0);
    --warning: oklch(0.65 0.18 85);

    /* ─── Border / Input ─── */
    --border: oklch(0.82 0.02 280);             /* 보라 틴트 테두리 (고대비) */
    --input: oklch(0.85 0.01 280);
    --ring: oklch(0.52 0.20 280);

    /* ─── Radius ─── */
    --radius: 0.75rem;                          /* base = 12px (시니어 더 둥글게) */
  }
}
```

## **10\. 타겟 맞춤 UX 전략**

### **타겟 사용자 프로파일**

| 항목 | 내용 |
| ----- | ----- |
| **연령** | 60\~80세 시니어 |
| **디지털 친숙도** | 낮음 (스마트폰 보급은 됐으나 앱 사용 경험 제한적) |
| **주 사용 맥락** | 아침 커피 시간, 저녁 취침 전 (5\~15분 짧은 세션) |
| **주 기기** | 아이패드 mini, 갤럭시탭 A (큰 화면 선호) |
| **주요 UX 장벽** | 버튼이 너무 작아 오클릭 / 한 화면에 정보 과다 / 시간 압박에 불안 / "실수하면 안 된다"는 심리적 부담 |

### **UX 전략**

1. **원 스텝 - 원 화면 원칙**: 한 화면에 하나의 핵심 액션만. 다음 버튼을 눌러야 다음 정보가 나옴
2. **실수 허용 설계**: 오답 시 즉각 정답을 알려주고 "괜찮아요, 다시 해봐요" 메시지. 절대 점수 감점 표시 금지
3. **느린 리듬 인터랙션**: 전환 속도 0.4\~0.5s. 카운트다운은 3초 이상. 급박한 사운드/효과 금지
4. **성취감 우선 피드백**: 틀린 것보다 맞은 것을 먼저 크게 표시. "오늘 7개 맞췄어요!" 방식
5. **반복 패턴 일관성**: 매일 같은 화면 순서, 같은 버튼 위치. 레이아웃 변동 금지
6. **이전/다음 명확화**: 뒤로가기 항상 좌상단 고정. "다음" 버튼은 항상 우하단 고정

## **11\. 최종 품질 검수 항목**

### **기능 QA (P0 기준)**

- \[ \] 오늘의 훈련 시작 → 훈련 완료 전체 플로우 5번 반복 오류 없음
- \[ \] 기억 카드 정답/오답 피드백 — 오답 시 빨간색 강조 과하지 않음
- \[ \] 훈련 중단 후 재진입 시 이어하기 또는 처음부터 선택 제공
- \[ \] 세션 완료 시 축하 애니메이션 + "오늘 N개 맞췄어요!" 메시지 표시

### **시니어 UX QA**

- \[ \] 모든 버튼 탭 영역 64px 이상 (디바이스 탭 테스트)
- \[ \] 인터랙티브 요소 간격 `gap-3`(12px) 이상 — 실제 기기에서 오클릭 테스트
- \[ \] 훈련 카드 텍스트 `text-2xl` 이상 (실제 기기 60세 사용자 가독성 확인)
- \[ \] 오클릭 후 실수 취소 가능한가? (정답 선택 전 되돌리기 가능)

### **접근성 QA**

- \[ \] Contrast Checker로 `--foreground(0.15)` 기반 텍스트 대비율 7:1 확인
- \[ \] 카드 정답/오답 구분 — 색상 외 아이콘(✓ / ?) 병행 표시 확인
- \[ \] VoiceOver / TalkBack으로 훈련 플로우 전체 접근 가능 확인

### **AI 생성 UI 오류 패턴 체크**

- \[ \] `text-xs`, `text-sm` 과다 사용 없음 (기억생생 금지)
- \[ \] `size-sm` 버튼 사용 없음
- \[ \] 인터랙티브 요소 간격 `gap-1`~`gap-2` 사용 없음 (최소 `gap-3`)
- \[ \] 애니메이션 duration 0.15s 이하 없음 (최소 0.2s)
